1. The Basics of the Web and HTML

The World Wide Web is just a collection of files hosted on servers (computers). These files are interpretted by web browsers that know how to make sense of HTML/CSS.

So... What does that mean? Something like this:

Breakin' It Down

  1. HTTP Get Request is what happens when you point your web browser at a web page. I.e. typing udacity.com in your address bar on your web broswer will send an HTTP Get Request to the server that hosts Udacity.
  2. HTTP Response Message is the server's response to your web browser saying, "Hey, gimme that webpage!" The server replies by sending your web browser a file that contains the information about how to display the webpage you're wanting.
  3. Compose & Render is just the web browser's attempt at displaying the file it now has in it's possession. It does this by interprettin the HTML/CSS in the file.

More on Taggin'

  • HTML Tags come in the format of an opening bracket (less than sign) "<" and a closing bracket (greater than sign) ">". They also come in two varieties:
    1. Container Tags - Come in an opening and closing pair. Want bold text? Just surround the text with the <strong> and </strong> tags and everything contained within the two will be bolded.
    2. Stand Alone Tags - These tags stand alone meaning they don't need a closing tag. Want an image on your page? Use the <img> tag. No closing tag necessary.

Taggin' with Style

  • Inline tags are what I've made liberal use of in these notes. Every time you see text that is bold or italicized it's because I used an inline tag to literally alter it in the line it's on. Generally speaking it's ok for one-off and instructional use cases, but over-use might lead the authorities to think you're a kidnapper. So, as a general rule, try to leave inline tags with the skates. In the 90's.
  • Block tags are how you make your pages sexy in the sense that newspapers are sexy. That is to say, if you look at a newspaper it has sextions. Want a new paragraph? Use the the <p> tag. Want a whole new section, like a sidebar? Use the <div> tag.

3. Adding Style to Structure with CSS

What Does CSS Look Like?

CSS has the format of:

tag .class #id {
    attribute1: value;
    attribute2: value;
}

With this format "tag" would represent a tag you'd want to style, ".class" would be a class name that you're defining and can apply to tags in your HTML, and "#id" would be a unique identifier for a unique element on your page.

What Is This Cascading Business?

Want more than black text on a white background? CSS is king. Using iterative coding, saving, and refreshing CSS will allow you use and abuse your webpage until it looks almost the way you'd like.

So, you have the temerity to ask about the meaning of cascading the context of styling sheets? Basically style can be applied in three ways: from a separate .css file, in the <head> section of your html file, or inline with the element in the html. This setup works the way Tea Partiers think government should: power "cascades" down. This means that the closer the styling is to the element, the more preference given to the style. The Power of Laziness (Thou Shalt Not Repeat Thyself)

What if you had a bunch of elements on your webpage that you wanted styled similarly? You could go edit the styles of each section individually. However, I'm trying to avoid carpal tunnel. So, I throw that style in my CSS file and then apply it wherever I please with the efficiency of a benevolent dictator.
